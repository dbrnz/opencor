The following tools are required in order to develop, build, test, run and package OpenCOR:

  • Git: a distributed version control system;
  • CMake 3.0 or later: a cross-platform build system;
  • C++ toolchain: a set of C++ tools. The toolchain is different depending on the platform you are using:
    • Windows: Visual Studio Express 2013 with Update 3 for Windows Desktop;
    • Linux: the g++ and libgl1-mesa-dev packages in Ubuntu; and
    • OS X: Xcode 5.1.1 (you will also need to get both the command line tools and PackageMaker, which can be done via the Xcode | Open Developer Tool | More Developer Tools... menu in Xcode).
  • Qt 5.3.1: this combines Qt 5.3.1 (a cross-platform application and UI framework) and Qt Creator 3.2.0 (its supporting Qt IDE):
    • Windows (you want to install the OpenGL version, so that OpenCOR can also work on Windows XP);
    • Linux 32-bit/64-bit (you might want to sudo install Qt under /opt rather than your home folder); and
    • OS X (you might want to install Qt under /Applications rather than your home folder).
  • NSIS 2.46 (Windows only): a program for creating Windows installers.

Finally, you might want to update your PATH environment variable by adding the following path(s):

  • Windows: C:\Program Files\CMake\bin (or C:\Program Files (x86)\CMake\bin, on 64-bit Windows), C:\Qt\5.3\msvc2013_opengl\bin (or C:\Qt\5.3\msvc2013_64_opengl\bin, on 64-bit Windows) and C:\Qt\Tools\QtCreator\bin;
  • Linux: /opt/Qt/5.3/gcc/bin (or /opt/Qt/5.3/gcc_64/bin, on 64-bit Linux) in /etc/environment on Ubuntu; and
  • OS X: /Applications/Qt/5.3/clang_64/bin in /etc/profile.
